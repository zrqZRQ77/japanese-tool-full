# Yomeru Design System v2

本文件是 Yomeru 网站的可执行设计规范。新增页面、组件或视觉调整时，先遵守本文件，再进入具体页面实现。

配套文件：

- `DESIGN.md`：长版设计原则与页面模板。
- `frontend/typography.css`：正式字体、字号、行高和字重 token。
- `frontend/design-system.css`：正式颜色、组件和页面视觉规则。
- `frontend/yomeru-ui-kit.html`：可直接打开的可视化组件与组合预览。

## 1. 产品视觉定位

Yomeru 是一个安静、清晰、适合长期使用的日语阅读工作台，不是营销落地页，也不是游戏化学习应用。

核心原则：

1. 阅读内容优先，装饰退后。
2. 紫色用于品牌、主行动、选中态和焦点，不做大面积背景。
3. 每个流程只有一个最突出的主行动。
4. 页面通过留白、分割线和 surface 层级组织，不依赖重阴影。
5. 同一屏中的按钮、卡片、输入框和标签必须使用同一套材料、圆角和字号。
6. 不新增临时颜色、临时字号或临时组件。

## 2. 字体与语言角色

### 中文界面

```css
--type-font-ui-zh:
  'Inter', 'PingFang SC', 'Microsoft YaHei', 'Noto Sans SC', sans-serif;
```

用于：中文导航、按钮、表单、说明、设置、统计和系统反馈。

### 日文界面控件

```css
--type-font-ui-ja:
  'Noto Sans JP', 'Hiragino Sans', 'Hiragino Kaku Gothic ProN',
  'Yu Gothic', 'Meiryo', sans-serif;
```

用于：日语界面按钮、导航、短标签和小尺寸工具文字。

### 日语学习内容

```css
--type-font-content-ja:
  'Noto Serif JP', 'Hiragino Mincho ProN', 'Yu Mincho', 'MS Mincho', serif;
```

用于：阅读正文、假名标注、词语详情、语法例句、打字答案和复述原文。

### 使用规则

- 中文 UI 不使用明朝体。
- 日语长文和学习示例优先使用明朝体。
- 日语小按钮和导航仍使用无衬线体，保证小字号清晰。
- 不在单个组件里重新声明字体；统一引用 typography token。

## 3. 字号系统

### 核心字号阶梯

| Token | 大小 | 用途 |
| --- | ---: | --- |
| `--type-size-ruby` | 10px | 假名、ruby |
| `--type-size-micro` | 11px | 极小状态和技术信息 |
| `--type-size-caption` | 12px | 标签、角标、小标题 |
| `--type-size-meta` | 13px | 元信息、辅助说明 |
| `--type-size-control` | 14px | 按钮、输入框、筛选 |
| `--type-size-body` | 15px | UI 正文、卡片正文 |
| `--type-size-emphasis` | 16px | 强调正文、关键值 |
| `--type-size-body-large` | 17px | 大正文、引导信息 |
| `--type-size-subtitle` | 18px | 卡片标题、小标题 |
| `--type-size-lead` | 19px | 引导句、重点信息 |
| `--type-size-section-title` | 20px | 区块标题、阅读正文 |
| `--type-size-feature-title` | 24px | 功能标题、较大数字 |
| `--type-size-page-title` | 28px | 页面标题 |
| `--type-size-display` | 32px | 展示标题 |
| `--type-size-hero` | 36px | 首页主标题 |
| `--type-size-hero-large` | 44px | 大屏展示标题 |
| `--type-size-hero-max` | 56px | 极少数品牌展示 |

### 行高

| Token | 值 | 用途 |
| --- | ---: | --- |
| `--type-line-compact` | 1.3 | 标题、标签 |
| `--type-line-ui` | 1.5 | 按钮、输入、导航 |
| `--type-line-body` | 1.7 | 中文说明、卡片正文 |
| `--type-line-reading` | 2 | 日语正文、假名阅读 |

### 字重

- Regular：400
- Medium：500
- Semibold：600
- Bold：700

### 排印规则

1. 普通 UI 正文统一 15px。
2. 按钮和输入控件统一 14px。
3. 面板标题统一 18px，页面标题统一 28px。
4. 日语阅读正文默认 20px，假名默认 10px。
5. 不在 `typography.css` 之外新增固定 `px` 字号。
6. `npm run check` 会阻止新的硬编码字号进入正式页面。
7. 旧的精细阶梯仅用于保持既有响应式比例，不作为新组件的首选。

| On primary container | `#21005D` | 浅紫底文字 |
| Secondary | `#625B71` | 次要强调 |
| Secondary container | `#E8DEF8` | 次按钮、辅助状态 |

### Surface 与文字

| Token | 色值 | 用途 |
| --- | --- | --- |
| Background | `#FFFBFE` | 页面背景 |
| Surface | `#FFFFFF` / `#FFFBFE` | 正文、主内容面 |
| Surface low | `#F7F2FA` | 低强调卡片、输入区 |
| Surface container | `#F3EDF7` | 工具栏、浮层 |
| Surface high | `#ECE6F0` | 强分组控件 |
| Main text | `#1C1B1F` | 主文字 |
| Secondary text | `#49454F` | 次要文字 |
| Muted text | `#6D6875` | 元信息、提示 |
| Border | `#CAC4D0` | 默认边框和分割线 |

### 学习状态

| 语义 | 前景 | 浅底 | 用途 |
| --- | --- | --- | --- |
| 品牌/新收藏 | `#6750A4` | `#EADDFF` | JLPT、收藏、当前状态 |
| 语法提示 | `#7D5260` | `#FFD8E4` | 助词、语法重点 |
| 待复习 | `#8A4B00` | `#FFDDBA` | 犹豫、提醒、需复习 |
| 错误/危险 | `#BA1A1A` | `#FFDAD6` | 真实错误、删除确认 |
| 已掌握 | `#2E6B47` | `#D9F2E3` | 记住了、完成 |

### 颜色规则

- 同一屏只允许一个深紫主按钮。
- 正文和词汇主标题使用深灰，不使用品牌紫。
- 紫色只表达品牌、选中、焦点和可行动状态。
- warning 表示学习压力；error 只表示真实错误或危险操作。
- 普通卡片边框使用中性灰，不使用紫色边框。

## 5. 间距、圆角与阴影

### 间距

采用 8px 节奏：`4 / 8 / 12 / 16 / 24 / 32 / 48 / 64`。

- 图标与文字：8px
- 按钮组：8–12px
- 表单项：12–16px
- 卡片内边距：16–24px
- 主模块间距：24–32px
- 页面大区块：48–64px

### 圆角

| 类型 | 建议值 |
| --- | ---: |
| 标签、标注 | 4–6px |
| 按钮、输入框 | 8px |
| 卡片、面板 | 12–16px |
| 弹窗、抽屉 | 16px |
| 胶囊 | 999px，仅用于状态标签 |

### 阴影

- 普通卡片默认无阴影或极轻阴影。
- 浮层和弹窗才使用阴影表达层级。
- hover 不上浮、不放大、不增加强阴影。
- 优先使用边框、surface 和留白建立层级。

## 6. 按钮系统

### 主按钮 `.btn-primary`

- 高度：至少 44px
- 字号：14px
- 字重：600
- 背景：Primary
- 文字：白色
- 用途：当前流程最重要的动作
- 同一可见流程最多一个

### 次按钮 `.btn-secondary`

- 高度：至少 40–44px
- 字号：14px
- 背景：白色或浅 surface
- 边框：1px 中性边框
- 用途：辅助动作、取消、返回

### 文字按钮 `.btn--text`

- 无填充、无重边框
- 用于上传文件、使用示例、查看全部等低优先级动作

### 图标按钮

- 点击区域至少 40×40px，移动端 44×44px
- 图标视觉尺寸 18–20px
- 必须有 `aria-label`
- 删除、关闭、朗读等操作保持同一图标粗细

### 危险按钮

- 只用于删除、清空、覆盖恢复
- 使用 error 文字或浅 error 底
- 不使用普通主按钮样式

## 7. 输入与表单

### 输入框

- 高度：44–50px
- 字号：14px
- 边框：1px 中性边框
- 圆角：8–12px
- 背景：白色或 Surface low
- Focus：紫色边框 + 轻量 focus ring

### 大文本输入

- 最小高度：120px
- 内容区像纸面，不做营销卡片
- 操作按钮放在同一视觉组中

### 表单组合

```text
标签
输入控件
必要的短状态文字
保存按钮
```

- 标签与输入框左边缘对齐。
- 不为每个字段增加长说明。
- 手机端纵向堆叠，桌面端可横向对齐。

## 8. 标签、状态与筛选

### 标签

- 字号：12px
- 高度：24–30px
- 圆角：6px 或真正的胶囊
- 只表达分类或状态，不做装饰

### 筛选按钮

- 默认透明或白底
- 当前项使用浅紫底和紫色文字
- 同一筛选组只保留一个选中态
- 不使用多个深色填充按钮

### 状态表达

状态必须同时包含文字，不只依赖颜色。

示例：

- `N5`
- `已收藏`
- `待复习`
- `没记住`
- `已掌握`

## 9. Surface 与卡片

### 页面级 Surface

只提供页面宽度、背景和留白，不画明显边框，不作为大卡片。

### 模块级 Surface

用于真正独立的工具，例如：

- 阅读面板
- 详情面板
- 设置分组
- 闪卡舞台
- 导出弹窗

允许 1px 边框和轻量圆角，默认不使用重阴影。

### 对象级 Surface

用于列表项、资料项、收藏项。外层已经有模块时，对象级 surface 应弱化为分割线或浅底，不再卡片套卡片。

## 10. 标准组件

| 组件 | 主要结构 | 关键规则 |
| --- | --- | --- |
| 工作栏 | 搜索/输入 + 一个主按钮 + 轻状态 | 不放多个主按钮 |
| 阅读工具栏 | 紧凑工具按钮组 | 工具栏必须弱于正文 |
| 详情面板 | 词语 + 读音 + 标签 + 释义 + 操作 | 不使用左侧装饰竖线 |
| 列表行 | 主信息 + 解释 + 状态 + 轻操作 | 内容优先于按钮 |
| 空状态 | 一句说明 + 一个轻按钮 | 不放插画和长教程 |
| 折叠管理区 | summary + 低频操作 | 默认折叠 |
| 闪卡舞台 | 进度 + 单卡 + 评分 | 一屏一项任务 |
| 设置项 | 标题/说明 + 控件/按钮 | 行式排列，危险操作分组 |
| 弹窗 | 标题 + 内容 + 底部操作 | 主按钮靠右，Esc 可关闭 |

## 12. 响应式

### 桌面

- 最大内容宽度：1180px
- 侧边导航：约 224px
- 阅读页：主内容 + 详情双栏

### 平板

- 详情面板下移或变为折叠区
- 多列组件变为两列或单列

### 手机

- 顶部品牌栏 + 底部主导航
- 所有触控目标至少 44px
- 表单和按钮纵向排列
- 优先减少内边距，不先缩小阅读字号
- 不允许页面横向溢出

验收尺寸：`1440×900 / 1280×720 / 430×932 / 390×844 / 320px`。

## 13. 可访问性

1. 键盘焦点清楚可见。
2. 图标按钮有可访问名称。
3. 状态不能只靠颜色表达。
4. 触控目标至少 44px。
5. 弹窗、菜单和抽屉支持 Esc 关闭。
6. 文本放大和窄屏时不重叠。
7. 动画尊重 `prefers-reduced-motion`。

## 14. 实现与验收

### 正式 CSS 顺序

```html
styles.css
design-system.css
grammar-layout.css
typography.css
```

`typography.css` 最后加载，作为字体和字号的最终规则。

### 新增 UI 流程

1. 选择已有页面模板。
2. 复用标准组件。
3. 必须新增组件时，先更新本规范和 UI Kit。
4. 在 `frontend/yomeru-ui-kit.html` 中验证。
5. 再进入正式页面。
6. 运行维护检查和视觉验证。

### 提交前检查

- 是否只有一个主操作最突出？
- 是否使用统一颜色 token？
- 是否使用统一字体和字号 token？
- 是否出现卡片套卡片？
- 是否新增了无意义紫色或重阴影？
- 桌面与手机是否无横向溢出？
- UI Kit 与正式页面是否一致？
