# Yomeru UI Kit v1

本文件是 Yomeru 页面统一视觉和组件组合的执行标准。它不是灵感板，而是后续改页面时要套用的最小组件库。

目标：让不同页面放在一起时像同一个产品，而不是每页各自好看。

## 总原则

1. 页面优先是工作台，不是营销页。
2. 每个页面只保留一个主要行动。
3. 高频功能露出，低频功能折叠。
4. 组件靠边框、留白、层级组织，不靠大色块和重阴影。
5. 紫色只用于主行动、当前状态、焦点和少量学习状态。
6. 说明文案能删就删，保留的文字必须帮助用户下一步操作。
7. 不做卡片套卡片。

## 1. 工作栏

用途：页面顶部的主要操作区。

组成：
- 左侧：搜索框、输入框或当前任务入口。
- 右侧：一个主按钮。
- 下方：轻量状态文字。

适用页面：
- 生词本
- 语法本
- 资料页
- 学习历史

不要：
- 同时放多个主按钮。
- 把统计做成 dashboard 卡片。
- 放大段说明文字。

## 2. 列表

用途：承载可扫描、可管理的数据。

结构：
- 顶部：列表标题 + 轻量筛选。
- 中间：列表行。
- 底部：低频管理入口。

不要：
- 每个列表项都做成独立重卡片。
- 在列表上方堆多个说明模块。
- 给普通 hover 加上浮和强阴影。

## 3. 列表行

用途：生词、语法、资料、历史记录等重复项。

推荐结构：
- 左侧：主信息。
- 中间：解释、来源或摘要。
- 右侧：状态 + 轻量操作。

状态：
- 掌握/认识：紫色。
- 待复习/犹豫：黄色。
- 没记住/错误：红色。

不要：
- 做成后台表格那样很多等权列。
- 操作按钮比内容更显眼。
- 同一行出现过多标签。

## 4. 空状态

用途：没有内容时给用户一个自然下一步。

结构：
- 一句短文字。
- 一个轻按钮。

示例：
- `还没有收藏的词。`
- `去阅读`

不要：
- 写教程式长说明。
- 放插画、大卡片或多个按钮。

## 5. 折叠管理区

用途：导出、清空、备份、恢复等低频功能。

规则：
- 放在主要内容后面。
- 默认折叠。
- 危险操作保持低频、明确。

不要：
- 把导出、清空放在页面最上方。
- 和主行动并列。

## 6. 详情面板

用途：显示当前选中内容的解释、读音、例句、来源。

结构：
- 简短标题。
- 主要解释。
- 次要信息。
- 一个相关行动。

不要：
- 标题重复页面名。
- 同时显示多个互相竞争的按钮。
- 填充大段说明文字。

## 7. 闪卡舞台

用途：专注复习。

结构：
- 顶部：轻量进度 + 退出。
- 中间：一张学习卡片。
- 底部：评分按钮。

规则：
- 进入复习时隐藏列表和管理区。
- 卡片保持纸面感，不用重阴影。
- 评分按钮用轻边框，hover 时再显示状态底色。

不要：
- 复习区和列表区同时抢注意力。
- 使用大面积红/黄/紫填充。

## 8. 表单区

用途：设置、导入、筛选、资料添加。

结构：
- 标签。
- 输入控件。
- 必要状态。
- 主按钮或保存按钮。

规则：
- 表单项垂直对齐。
- 输入框宽度一致。
- 状态文字短。

不要：
- 每个字段都写长说明。
- 按钮散落在不同位置。

## 页面模板

### 列表工作台

```text
工作栏
轻状态
列表标题 + 筛选
列表
折叠管理区
```

用于：生词本、语法本、资料、历史。

### 专注任务页

```text
轻量顶部
主任务区域
状态 / 操作
```

用于：阅读、闪卡、打字练习。

### 设置页

```text
表单区
分组操作
低频危险操作
```

用于：设置与帮助、备份恢复。

## 后续改页面的检查表

每次改页面前检查：

1. 页面是否只有一个主行动？
2. 搜索/输入是否在最自然的位置？
3. 低频功能是否折叠或下移？
4. 统计是否只是轻状态？
5. 列表行是否以内容为主？
6. 空状态是否短且有下一步？
7. 组件是否来自本 UI Kit，而不是临时发明？
