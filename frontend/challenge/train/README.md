# Yomeru 电车日语输入挑战：A3 传播功能

## 当前实现

- 新宿到上野的山手线短路线，共 13 个连续站点。
- 开始页只保留标题、原创半环路线和三个核心控件；重复说明、成功状态文案和冗长页脚已删除或压缩。
- 模式、提示与发车使用受限宽度控件，电脑和手机开始页无需滚动即可操作。
- 两种模式：汉字→假名、假名→汉字。
- 可随时开关“练习提示”，显示当前站答案并记录使用提示的站数；每一局默认关闭。
- 纯挑战和使用提示的练习成绩分别保存，不混用最佳纪录。
- 点击“发车”后才开始计时，输入框使用 `preventScroll` 自动聚焦。
- 输入使用原生表单、`enterkeyhint="done"` 和明确的“确认”按钮；手机不再只依赖键盘 Enter。
- 正确提交后不禁用输入框，确认按钮不夺取焦点，iPhone 键盘可在连续 13 站之间保持打开。
- 日语 IME composition 期间不会提交；先确认候选，再按完成键或点击“确认”。
- 假名→汉字模式下，开始页路线继续使用站号避免答案泄露；答题页左侧显示序号、圆点和作为题面的平假名。汉字→假名模式则显示汉字站名。
- 汉字路线标签与圆点保持统一距离，长站名使用一致的两行基线。
- 严格答案判断、错误后继续修改、正确后只前进一站；常见中文输入字形使用人工白名单，接受后提示日文标准站名。
- 实时用时、进度、正确率、连续正确数和路线位置。
- 输入页使用与开始页一致的 880px 外层卡片和正式网站 `assets/brand/yomeru-mark.svg` Logo。
- 桌面答题区由约 205px 路线栏和单一主答题卡组成；题目和 590px 输入框居中，答案开关与答案标签收纳在卡片底部。
- 当前/下一站默认显示当前模式的题面；开启“显示答案”后显示完整的汉字与读音。
- 桌面路线完整展示 13 站，列车图标缩小并使用独立右侧通道，不遮挡圆点、序号或站名。
- 手机统计栏为单行，路线使用完整 13 点全局进度，题目、输入和辅助设置可在首屏完成。
- 结果页压缩字号和按钮间距，并明确区分纯挑战、部分提示和全程提示。
- “平均每站”使用总用时 ÷ 13；结果页用“错误次数”替代容易误解的 CPM。
- 独立 localStorage 保存最佳成绩、最近 5 次、总挑战次数和最近难度。
- 1080×1350 竖屏 PNG 成绩卡，列车在完成后停靠上野终点；练习模式使用更醒目的标记和独立分享文案。
- Preview 分享当前部署地址；正式域名环境使用 `https://yomeru.japanese-hub.com/challenge/train`。
- 390px、430px 和桌面响应式布局。

## 成绩卡与分享

成绩卡包含：

- Yomeru 品牌与挑战名称。
- 新宿到上野原创简化路线。
- 完成时间、模式、正确率、平均每站、错误次数和最佳连续。
- 挑战页面链接。
- 非铁路运营机构官方产品声明。

分享顺序：

1. 设备支持文件分享时，分享 PNG 成绩卡。
2. 仅支持普通 Web Share 时，分享文字与挑战链接。
3. 不支持系统分享时，复制挑战链接。
4. 剪贴板也不可用时，自动下载 PNG 成绩卡。
5. 用户取消系统分享不视为错误。

## 答案与 IME 规则

1. `compositionstart` 到 `compositionend` 期间不允许 Enter 提交。
2. 同时检查 `KeyboardEvent.isComposing`，避免 Safari 和移动端误提交。
3. 比较前执行首尾空格清理、全角/半角空格移除和 Unicode NFKC 规范化。
4. 汉字→假名模式只接受 `acceptedKana` 中人工列出的平假名。
5. 假名→汉字模式只接受 `acceptedKanji` 中人工列出的写法。
6. 不做编辑距离、同音字或宽松模糊匹配。
7. 正确提交后的短暂移动锁定期间，连续 Enter 不会跳过站点。
8. 错误答案不清空输入，也不会结束游戏。

## 计时与计分规则

- 点击“发车”时记录 `startedAt`，此前计时保持为 0。
- 使用 `performance.now()` 计算真实经过时间；切到后台再回来仍按经过时间计算。
- 第一版不提供暂停键；纯挑战和练习模式都保持连续计时，避免误解成绩规则。
- 记录 `correctSubmissions`、`wrongSubmissions`、`stationTimes[]`、`bestStreak` 和 `correctChars`。
- `accuracy = correct / (correct + wrong)`。
- `cpm = correctChars / elapsedMinutes`。
- 成绩比较顺序：正确率更高 → 完成时间更短 → CPM 更高。
- 最后一站只能触发一次结算。
- 没有真实服务端统计时不显示用户百分位。

## 本地数据边界

唯一命名空间：`yomeru_train_typing_v1`。

```json
{
  "schemaVersion": 1,
  "bestByMode": {},
  "recentResults": [],
  "totalChallenges": 0,
  "lastMode": "kanji-to-kana",
  "lastShowHints": false
}
```

只保存最佳成绩、最近 5 次、总挑战次数、最近难度和提示开关。纯挑战使用原模式键，练习模式使用 `:practice` 后缀保存独立纪录。损坏或旧格式数据会安全回退为空状态。

禁止读取或覆盖：

- `reading_vocab_list`
- 阅读历史
- 生词复习状态
- Yomeru 完整备份数据
