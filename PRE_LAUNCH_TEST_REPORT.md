## 1. 上线判断原则

本项目达到以下条件时，可以作为 MVP 上线：

1. 用户能够完成一次完整学习闭环：输入文章 → 阅读 → 查看单词 → 收藏生词 → 生词本 → 闪卡复习 → 学习记录。
2. 核心功能失败时不会白屏、卡死或造成数据丢失。
3. Chrome、Safari 和 iPhone Safari 中可以完成核心流程。
4. Vercel 线上环境第一次访问时，本地词典、CSS 和 JavaScript 能正常加载；远程 Kuromoji 不属于当前 MVP 启用能力。
5. 没有 P0 或 P1 级问题。
6. P2 问题可以存在，但必须不阻断核心流程，并记录在上线后优化清单中。

### 问题等级

| 等级 | 定义 | 是否阻止上线 |
|---|---|---:|
| P0 | 网站打不开、白屏、数据严重损坏、安全问题 | 是 |
| P1 | 核心学习流程无法完成，文件导入或阅读主功能不可用 | 是 |
| P2 | 功能可以使用，但有明显体验、兼容或提示问题 | 视数量决定 |
| P3 | 视觉细节、文案、非核心体验优化 | 否 |

### 测试状态

| 状态 | 含义 |
|---|---|
| PASS | 已测试并通过 |
| FAIL | 已测试但失败，需要修复 |
| BLOCKED | 因环境或依赖问题暂时无法测试 |
| PENDING | 尚未测试 |
| N/A | 当前 MVP 不测试或不适用 |
